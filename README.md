# HideProcess

`HideProcess` 是一个用于 Windows 的老板键工具。它可以把你选中的程序窗口立即隐藏，而不是最小化；隐藏后任务栏中不会保留图标，并且可以通过预设热键恢复显示。

## 主要功能

- 隐藏和恢复指定程序窗口
- 支持全局热键，也支持隐藏键和显示键相同的 `Toggle` 模式
- 支持三键及以上组合键，例如 `Ctrl + Shift + X`
- 支持分组管理目标程序
- 每个分组可单独设置隐藏/显示热键
- 支持直接拖拽程序卡片进行分组、移出分组
- 支持窗口选择模式，直接用鼠标点选正在运行的窗口加入目标
- 支持隐藏时静音，并可对每个目标单独配置
- 支持导入 / 导出配置
- 支持开机启动、关闭时最小化到托盘、单实例运行
- 支持自动检查更新和手动检查更新
- 支持中文 / English

## 分组模式

- 已选目标可以按组管理
- 未分组目标会直接显示在分组区域外
- 分组支持折叠、重命名
- 双击分组文件夹图标可展开 / 折叠
- 双击分组名称可直接进入编辑
- 拖拽程序卡片到分组即可加入该组
- 拖回未分组区域即可移出分组

## 热键说明

- 全局热键：对所有已添加目标生效
- 分组热键：仅对对应分组生效
- 全局热键和分组热键都允许留空
- 当同一组的隐藏热键和显示热键完全相同时，该组会自动进入 `Toggle` 模式
- 热键触发后会尽量阻止继续传递给系统或前台程序

## 使用方式

1. 在主界面选择运行中的程序并添加，或进入窗口选择模式直接点选窗口
2. 在主界面管理目标分组
3. 在设置中配置全局热键和分组热键
4. 需要时按下对应热键隐藏或恢复窗口

## 运行环境

- Windows 10 / 11
- x64
- .NET 8 运行环境仅在某些开发场景下需要；发布版已提供自包含构建

## 本地运行

```powershell
dotnet build HideProcess.sln
dotnet run --project .\HideProcess.App\HideProcess.App.csproj
```

## 正式构建

```bat
Build-Release.bat
```

## 已知限制

- 目标匹配主要基于进程名和进程路径，不是所有多窗口程序都能做到完全独立控制
- 浏览器等多标签应用目前仍以窗口为单位处理，而不是单个标签页
- 某些高权限或特殊渲染方式的窗口，隐藏 / 恢复行为可能受系统限制
